Notebooks

Notebooks

The notebooks in this repo contain plots and run scripts to generate the data for those plots. The contents of the notebooks are summarised here and contain links to the rendered notebooks are included.

In general, each directory contains a notebook and a collection of scripts. Since sudo access is needed to run MemSnoop, these scripts are stand-alone.

Note that these scripts should be run before the notebooks if trying to recreate the plots.

Basic Analysis

Basic analysis of the memory usage during the training of a simple CNN on a single CPU. The sampling window was 0.2 seconds. That is, the sampling procedure went something like this:

  1. Mark all the applications pages as idle.
  2. Run applition for 0.2 seconds
  3. Pause application
  4. Determine which pages are active and update datastructures.
  5. Repeat

CPU Analysis

Plots and some analysis of how the memory requirements for training (specifically for Tensorflow) scale as the number of available processors is increased.
